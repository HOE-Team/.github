## 🌟 关于我们

HOE Team 是由 [GoldenHoe](https://github.com/GoldenHoe) 与 [TNTyep520](https://github.com/TNTyep520) 共同创立的一个社区驱动的技术团体。我们秉承 **Harmony（和谐）、Openness（开放）、Equality（平等）** 的核心信条，致力于开发实用、高效的工具类应用程序。

我们的使命是通过代码解决实际问题，创造对社区有价值的工具，并在开放协作中共同成长。

## 🛠 技术栈

### Android 开发
- **语言**: Kotlin
- **UI框架**: Jetpack Compose
- **平台**: Android SDK & NDK
- **设计**: Material Design 3

### Python GUI 应用开发
- **GUI框架**: PyQt / PySide
- **Web框架**: FastAPI
- **数据处理**: Pandas, NumPy
- **其他**: 自动化脚本、后端服务与 API 开发

### 开发与协作
- **版本控制**: Git & GitHub
- **自动化**: CI/CD (GitHub Actions)

## 📂 项目展示

### 重点项目
- **[MCSEasy](https://github.com/HOE-Team/MCSEasy)** - 轻量、简洁的 Minecraft 服务器管理工具，帮助用户更简单地部署和管理服务器。
- **[OpenDroidChat](https://github.com/HOE-Team/OpenDroidChat)** - 适配 Android 6.0+ 的 LLM API 聊天客户端，提供流畅的 AI 对话体验。

*更多项目正在开发中，欢迎关注我们的 [组织主页](https://github.com/HOE-Team)！*

## 🤝 如何参与

我们欢迎所有对技术有热情、认同我们信条的开发者加入！

### 参与方式
1. **反馈与建议** - 使用 Issues 提出建议或报告问题
2. **代码贡献** - 查看带有 `good-first-issue` 或 `help-wanted` 标签的任务
3. **项目维护** - 协助文档编写、代码优化和测试
4. **创意分享** - 在 Discussions 中提出新的工具开发想法

### 贡献流程
1. Fork 目标项目仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request 并描述变更内容

## 📜 行为准则

在参与 HOE Team 社区时，请遵守我们的核心价值观：

- **Harmony 和谐** - 尊重不同观点，进行建设性讨论
- **Openness 开放** - 保持透明沟通，积极分享知识
- **Equality 平等** - 平等对待每位贡献者，无论经验水平

我们致力于提供一个友好、安全、包容的协作环境。任何形式的骚扰或不尊重行为都不会被容忍。
