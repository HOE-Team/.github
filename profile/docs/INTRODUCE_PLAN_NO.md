# 计划概述

**PLAN NO 简述**

PLAN NO 是 HOE Team 发起的技术计划，核心目标是复刻、优化并超越某些挂羊头卖狗肉、开源违规或闭源侵权的劣迹项目。

我们不针对"写得差"的开源项目，虽然差，但是他们接受审计，接受协作。我们针对的是三类行为：

1. **挂 GPL 却喊"我有权不开源"的虚假承诺**
2. **用 GPL 代码却闭源分发的规则无视**
3. **打包侵权工具、不标许可证、用户崩溃无人管的闭源糟粕**

PLAN NO 坚持用现代技术栈、遵守开源规则、提供透明合规的替代方案，**用代码说话，做本该更好的工具**。

---

### 我们的核心理念

**运用现代技术，遵守开源规范，打造透明合规的替代产品**

在开源生态中，我们致力于通过以下方式推动健康发展：
- **完全开源**：在明确的许可协议下开放所有源代码
- **透明开发**：代码仓库公开，开发过程可追溯
- **尊重版权**：明确标注所有依赖和资源的许可信息
- **社区驱动**：欢迎用户反馈、贡献代码和参与讨论

# 计划目标

| 关注领域 | 现状挑战 | PLAN NO 解决方案 |
|----------|----------|------------------|
| 许可证一致性 | 声明开源但实际闭源 | 在许可协议下完全开源，代码可供审查 |
| 代码透明度 | 使用开源代码但闭源分发 | 遵守所有依赖许可，代码完全公开 |
| 工具分发 | 打包侵权工具未标注许可 | 不打包第三方工具，提供使用指南 |
| 责任明确 | 无证无责，用户困惑无人管 | 明确许可条款，提供 Issue 反馈渠道 |

# 计划成果

### NNETB (Windows 平台)
- **平台**：Windows
- **技术栈**：Kotlin + Compose Multiplatform
- **许可证**：MIT
- **状态**：Beta，可用性验证阶段

### NFL (NNETB For Linux)
- **平台**：Linux
- **技术栈**：Kotlin + Compose Multiplatform + JAR 分发
- **许可证**：MIT
- **特点**：自动检测包管理器（APT、DNF、PACMAN 等）
- **状态**：Alpha，初期开发阶段
